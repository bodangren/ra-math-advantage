# Module 5 Class-Period Plan

This document is the audited class-period breakdown for Module 5.

It should be treated as the canonical planning source for Module 5 class periods unless a later audit supersedes it.

It follows the planning model in `curriculum/course-spec.md`:

- the **class period** is the atomic planning unit
- `instruction` days must map to `conductor/course-objectives.md`
- `instruction` days must name the specific worked examples they use
- `mastery`, `jigsaw`, `review`, and `test` days are marked primarily by day type

## Notes

- This is a **planning model**, not a literal calendar.
- Module 5 contains 8 course objectives but 10 instruction days, so the extra instructional space is used to slow down the densest textbook lesson: geometric sequences and series.
- Lesson `5-4` is treated as a multi-period lesson bundle because it contains both sequence-generation and series-summation work.
- The `5e` objective is broad enough to carry the full `5-3` lesson in a single instruction day.
- This plan has been checked against the current repaired lesson source files and worked-example numbering.

## Objective Coverage Check

All Module 5 course objectives appear as a primary objective in the plan:

- `5a`, `5b`, `5c`, `5d`, `5e`, `5f`, `5g`, `5h`

## Why This Split Works

- `5-1` divides naturally into a growth day and a decay day.
- `5-2` separates equation solving from inequality solving even though the inequality objective uses a lighter source-example set.
- `5-3` is broad but coherent enough to function as one base-`e` instruction day.
- `5-4` is the densest lesson in the module and needs four class periods: two for sequence generation and two for series summation.
- `5-5` is compact and works well as a single modeling-and-regression day.
