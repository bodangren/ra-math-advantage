# Module 6 Class-Period Plan

This document is the audited class-period breakdown for Module 6.

It should be treated as the canonical planning source for Module 6 class periods unless a later audit supersedes it.

It follows the planning model in `curriculum/course-spec.md`:

- the **class period** is the atomic planning unit
- `instruction` days must map to `conductor/course-objectives.md`
- `instruction` days must name the specific worked examples they use
- `mastery`, `jigsaw`, `review`, and `test` days are marked primarily by day type

## Notes

- This is a **planning model**, not a literal calendar.
- Module 6 has 10 course objectives and 12 instruction days, so two objectives intentionally span more than one class period.
- The extra instructional space is used for:
  - an additional natural-logarithm application day in Lesson `6-4`
  - a two-day treatment of continuous growth modeling in Lesson `6-5`
- When a single source example contains multiple substantial parts, the plan may reuse that example across more than one class period with explicit part labels.
- This plan has been checked against the current repaired lesson source files and worked-example numbering.

## Why This Split Works

- `6-1`, `6-2`, and `6-3` each divide naturally into two objective-based instruction days.
- `6-4` needs three days because the natural-logarithm material includes both symbolic simplification and applied solving/model analysis.
- `6-5` needs three days because continuous growth modeling is substantial enough to span two class periods before decay applications are introduced.
- The plan stays within the original module-level instruction-day budget while preserving explicit objective/example alignment.
