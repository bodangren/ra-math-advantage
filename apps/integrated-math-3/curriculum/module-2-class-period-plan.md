# Module 2 Class-Period Plan

This document is the audited class-period breakdown for Module 2.

It should be treated as the canonical planning source for Module 2 class periods unless a later audit supersedes it.

It follows the planning model in `curriculum/course-spec.md`:

- the **class period** is the atomic planning unit
- `instruction` days must map to `conductor/course-objectives.md`
- `instruction` days must name the specific worked examples they use
- `mastery`, `jigsaw`, `review`, and `test` days are marked primarily by day type

## Notes

- This is a **planning model**, not a literal calendar.
- Most instruction days use **2-3 worked examples**.
- A few source examples support prerequisite recognition or broader analysis without mapping cleanly to a distinct Module 2 objective code.
- In those cases, the example is still included, but it is treated as supporting the day's main objective rather than defining a new one.
- This plan has been checked against the current repaired lesson source files and worked-example numbering.

## Source-Example Notes

Two source examples are intentionally retained as support examples rather than treated as stand-alone objective drivers:

- `2-2, Example 5 ‚Äî Find Average Rate of Change`
  This supports analysis of polynomial behavior, but it does not map to a distinct Module 2 objective code.
- `2-3, Example 1 ‚Äî Identify Polynomials`
  This supports readiness for polynomial operations, but the module objective list does not include a separate classification objective.

## Why This Split Works

- `2-1` divides naturally into a power-function day and a broader polynomial-analysis day.
- `2-2` separates zero approximation from extrema and function-feature analysis.
- `2-3` separates addition/subtraction from multiplication and applied modeling.
- `2-4` separates long division from synthetic division cleanly.
- `2-5` uses the same objective code across two days because Pascal's Triangle and the Binomial Theorem are better taught as separate instructional beats.
