# Module 3 Class-Period Plan

This document is the audited class-period breakdown for Module 3.

It should be treated as the canonical planning source for Module 3 class periods unless a later audit supersedes it.

It follows the planning model in `curriculum/course-spec.md`:

- the **class period** is the atomic planning unit
- `instruction` days must map to `conductor/course-objectives.md`
- `instruction` days must name the specific worked examples they use
- `mastery`, `jigsaw`, `review`, and `test` days are marked primarily by day type

## Notes

- This is a **planning model**, not a literal calendar.
- Module 3 has fewer worked examples than most modules, so the instruction spine is more concept-dense.
- Some instruction days use a smaller number of worked examples and rely more heavily on proof, discussion, or graph interpretation.
- This module divides cleanly into four clusters: graphing/factoring, identities, substitution/factor theorem, and roots/zeros/graphing.
- This plan has been checked against the current repaired lesson source files and worked-example numbering.

## Why This Split Works

- `3-1` is compact enough to function as a single graphing-based instruction day.
- `3-2` carries the heaviest algebraic load in the module, so it is split into factoring and quadratic-form days.
- `3-3` works best as a dedicated proof-and-reasoning day rather than being merged into a procedural lesson.
- `3-4` separates synthetic substitution from factor-theorem application cleanly.
- `3-5` divides naturally into a roots-theory day and a zeros-to-graphing synthesis day.
