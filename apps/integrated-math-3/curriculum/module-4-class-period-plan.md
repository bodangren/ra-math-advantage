# Module 4 Class-Period Plan

This document is the audited class-period breakdown for Module 4.

It should be treated as the canonical planning source for Module 4 class periods unless a later audit supersedes it.

It follows the planning model in `curriculum/course-spec.md`:

- the **class period** is the atomic planning unit
- `instruction` days must map to `conductor/course-objectives.md`
- `instruction` days must name the specific worked examples they use
- `mastery`, `jigsaw`, `review`, and `test` days are marked primarily by day type

## Objective Coverage Check

All Module 4 course objectives appear as a primary objective in the plan:

- `4a`, `4b`, `4c`, `4d`, `4e`, `4f`, `4g`, `4h`, `4i`, `4j`, `4k`, `4l`, `4m`

## Why This Split Works

- `4-1`, `4-2`, `4-3`, and `4-4` each divide naturally into two objective-based instruction days.
- `4-5` needs three instruction days because the course objective list separates simplification, operations, and rationalizing denominators into distinct outcomes.
- `4-6` divides cleanly into an algebraic-solving day and a graphing/systems day.
- `4-5` now has explicit worked examples for rationalizing denominators and conjugates, so `4k` can stand as a full instruction day rather than an inferred gap.
