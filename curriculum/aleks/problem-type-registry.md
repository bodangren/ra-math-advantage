# ALEKS-Style Problem-Type Registry

This registry names reusable problem families for Integrated Math 3. It is derived from the current course objectives, lesson worked examples, and audited class-period plans.

Status meanings:

| Status | Meaning |
|---|---|
| `planned` | Family is identified for curriculum mapping, but not implemented as an activity component yet. |
| `component-planned` | A Conductor component track already names or implies this family. |
| `composite` | Family is best represented by a bundle of smaller implemented families plus a synthesis prompt. |

## Implementation Notes

- Families that share the same interaction shape should reuse a single component with different authored definitions.
- Graph-heavy families should share the graphing explorer foundation, not duplicate canvases by module.
- Algebra-heavy families should share the step-by-step solver foundation, with family-specific step templates and misconception distractors.
- Statistics families need table, distribution, and interpretation artifacts preserved in `practice.v1` submissions.
- Modeling families should preserve both equation setup and interpretation because teacher review needs to see more than a final numeric answer.
