# Module 1 Class-Period Plan

This document is the audited class-period breakdown for Module 1.

It should be treated as the canonical planning source for Module 1 class periods unless a later audit supersedes it.

It follows the planning model in `curriculum/course-spec.md`:

- the **class period** is the atomic planning unit
- `instruction` days must map to `conductor/course-objectives.md`
- `instruction` days must name the specific worked examples they use
- `mastery`, `jigsaw`, `review`, and `test` days are marked primarily by day type

## Notes

- This is a **planning model**, not a literal calendar.
- Most instruction days use **2-3 worked examples**.
- A few objective-isolating days use **1 anchor example** plus guided practice, discussion, or error analysis.
- Some course objectives are treated as **embedded objectives** rather than the single primary objective for a day.
- This plan has been revised against the repaired Module 1 lesson source files and current worked-example numbering.

## Why This Split Works

- `1-1`, `1-6`, and `1-7` divide naturally into two class periods by objective cluster.
- `1-3` and `1-4` are dense enough that they work best as two instruction days plus a nearby mastery or jigsaw day.
- `1-5` now carries a heavier example load, but it still fits the current two-day split if the vertex-form day is taught through a tighter guided sequence.
- `1-8` now clearly splits into a linear-quadratic systems day and a quadratic-quadratic systems day, with intersection interpretation embedded across both.
- The non-instruction days create room for long-term mastery learning and mathematical-practices work without forcing a one-to-one match between textbook lessons and class periods.
